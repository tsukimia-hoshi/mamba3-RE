下面是按你最新口径整理后的 新版算子规划。
仓库里的 mamba3.py 原版大致是：in_proj 一次性投影出 [z, x, B, C, dd_dt, dd_A, trap, angle]，其中 dd_dt 和 dd_A 都是 token 级；A=-softplus(dd_A)，DT=softplus(dd_dt+dt_bias)；同时保留 trap、data-based RoPE angle、B/C RMSNorm。 你原始规划则已经把结构改成 chunk 级 scan：chunk 内聚合写入项 (U_m)，chunk 间只做一次状态递推。
至于MIMO/SISO，在这版里不再区分，如果非要理解，可以理解为：1、一个chunk只有一步，但有门控MIMO；2、或者也可以理解为抛弃了chunk内动力学的SISO。
新版算子：Chunk-level Delta/A + Token Write Gate Mamba3
0. 总体目标
新版算子仍然以 mamba3-RE 的 Mamba3 模块为主体，尽量保留原版的：
[
z,\ x/V,\ B/K,\ C/Q,\ D,\ B/C\ \text{bias},\ B/C\ \text{norm},\ MIMO/SISO\ \text{结构},\ out_proj
]
但针对高分辨率图像特征和低照 RAW 去噪，做以下核心改动：
1.	(\Delta) 从 token 级改为 chunk 级。
原版每个 token 生成 dd_dt；新版每个 chunk 生成一次 dd_dt_m。
2.	(\Delta) 由卷积生成的块级特征得到。
不再由 token 级 in_proj 直接给出。
3.	dd_A 也改为 chunk 级，由块级特征得到。
原版 dd_A 是 token 级；新版每个 chunk 只有一个 dd_A_m。
4.	不要 trap。
删除原版投影里的 trap 分支，kernel 里也不再传入 Trap。
5.	不要 data-based RoPE。
删除原版由 angle 分支生成的 data-based rotary angle，不维护 angle_state，不调用 data-dependent RoPE。
6.	chunk 内 token 使用独立写入门控。
不再 softmax，也不要求破坏 token 独立可加性。每个 token 自己生成写入 gate，类似原版 (\Delta) 的 logits → activation 方式。
7.	所有原版 norm 位置加开关。
可以保留原版 RMSNorm / RMSNormGated，也可以切换为 Derf。
________________________________________
1. 输入与分块
输入序列：
[
x_1,\ldots,x_T,\qquad x_t\in\mathbb R^{d_{\text{model}}}
]
按 chunk size (L) 分块：
[
\mathcal C_m={mL,\ldots,mL+L-1},
\qquad m=0,\ldots,M-1
]
对图像特征，可以把 2D feature map flatten 成序列，也可以按局部窗口/patch 形成 chunk。
________________________________________
2. Token 级投影：保留原版主体
原版 mamba3-RE 投影顺序是：
[
[z,\ x,\ B,\ C,\ dd_dt,\ dd_A,\ trap,\ angle]
]
新版改成：
[
[z,\ v,\ B,\ C,\ r]
W_{\text{tok}}x
]
其中：
[
v \equiv x_{\text{ssm}}
]
[
B \equiv K
]
[
C \equiv Q
]
[
r \equiv \text{token write-gate logits}
]
也就是说，原版里的 x 仍然作为 value/input 使用；B 和 C 仍然对应写入/读出方向；z 仍然作为输出 gate。
新版 token 级投影里 不再包含：
[
dd_dt,\quad dd_A,\quad trap,\quad angle
]
其中 dd_dt 和 dd_A 移到 chunk 级卷积分支；trap 和 angle 直接删除。
________________________________________
3. Chunk 级特征生成
对输入特征做 chunk-level convolution：
[
f_m=\operatorname{Conv}_{L,L}(x)_m
]
其中 kernel size 和 stride 都等于 chunk size：
[
\text{kernel size}=L,\qquad \text{stride}=L
]
这个 (f_m) 是当前 chunk 的块级特征，用来生成：
[
dd_dt_m,\qquad dd_A_m
]
可以写成：
[
dd_dt_m=W_{\Delta}f_m
]
[
dd_A_m=W_A f_m
]
或用一个 combined projection：
[
[dd_dt_m,\ dd_A_m]=W_{\text{chunk}}f_m
]
________________________________________
4. Chunk 级 (\Delta_m)
原版 mamba3-RE 中：
[
DT_t=\operatorname{softplus}(dd_dt_t+dt_bias)
]
新版改为 chunk 级：
[
\Delta_m
\operatorname{softplus}(dd_dt_m+b_{\Delta})
]
这里：
[
\Delta_m\in\mathbb R^{H}
]
或按实现需要扩展成：
[
\Delta_m\in\mathbb R^{H\times N}
]
最直接的版本是每 head 一个 (\Delta_m)。
chunk 内所有 token 共享同一个 (\Delta_m)。
________________________________________
5. Chunk 级 (A_m)
原版中：
[
A_t=-\operatorname{softplus}(dd_A_t)
]
并做 clamp：
[
A_t=\operatorname{clamp}(A_t,\ \max=-A_{\text{floor}})
]
新版改为 chunk 级：
[
A_m
-\operatorname{softplus}(dd_A_m)
]
[
A_m
\operatorname{clamp}(A_m,\ \max=-A_{\text{floor}})
]
所以离散衰减项为：
[
\Lambda_m
\exp(A_m\Delta_m)
]
或者按原版 kernel 变量命名：
[
ADT_m=A_m\Delta_m
]
关键变化是：
dd_A 不再是 token 级动态参数，而是由 chunk 卷积特征生成的块级动态参数。
________________________________________
6. 删除 trap
原版 mamba3-RE 里有：
[
trap
]
并传入 Mamba3 kernel。新版直接删除：
[
trap \quad \text{removed}
]
因此状态更新不使用 exponential-trapezoidal 修正项，只保留基础 chunk-level recurrence。
实现上对应：
•	in_proj 输出维度去掉 self.nheads 的 trap 部分；
•	forward split 不再 split trap；
•	kernel API 不再传 Trap=trap；
•	step/inference 路径也去掉 trap。
________________________________________
7. 删除 data-based RoPE
原版 mamba3-RE 中存在 angle 分支，并在 inference step 里调用 data-dependent rotary 相关逻辑。新版删除：
[
angle
]
[
Angles
]
[
angle_state
]
[
apply_rotary_qk_inference_fwd
]
即：
不再由输入数据生成 RoPE angle，也不在 SSM 的 (B/C) 或 (Q/K) 上使用 data-based RoPE。
如果后续需要位置编码，建议放在算子外部，例如普通 2D positional embedding、坐标 bias、窗口相对位置偏置等，而不是保留 Mamba3-RE 的 data-based RoPE 分支。
________________________________________
8. Chunk 内 token 独立写入门控
原始规划里 chunk 内写入使用 softmax 权重：
[
U_m
\sum_{i\in\mathcal C_m}
\alpha_{m,i}
v_{m,i}k_{m,i}^{\top}
]
新版改成 token 独立写入 gate。
每个 token 生成 gate logit：
[
r_{m,i}=w_r^\top x_{m,i}
]
然后类似原版 (\Delta) 的激活方式，得到正值写入门控：
[
g_{m,i}
\operatorname{softplus}(r_{m,i}+b_g)
]
如果你想让 gate 有界，也可以换成：
[
g_{m,i}=\sigma(r_{m,i})
]
chunk 写入项：
[
U_m
\frac{1}{L}
\sum_{i\in\mathcal C_m}
g_{m,i}
v_{m,i}k_{m,i}^{\top}
]
如果对应 mamba3-RE 记号：
[
k_{m,i}\equiv B_{m,i}
]
[
q_{m,i}\equiv C_{m,i}
]
[
v_{m,i}\equiv x_{m,i}^{\text{ssm}}
]
则：
[
U_m
\frac{1}{L}
\sum_{i\in\mathcal C_m}
g_{m,i}
x_{m,i}^{\text{ssm}}
B_{m,i}^{\top}
]
也不强行做 token 间竞争。chunk 内是非因果、独立可加写入；chunk 间由 SSM 动力学负责动态传播。
________________________________________
9. Chunk 级状态递推
对每个 head，状态为：
[
S_m
]
新版状态更新为：
[
\boxed{
S_m
\exp(A_m\Delta_m)S_{m-1}
+
\Delta_m U_m
}
]
即：
[
\boxed{
S_m
\Lambda_mS_{m-1}
+
\Delta_m
\frac{1}{L}
\sum_{i\in\mathcal C_m}
g_{m,i}
v_{m,i}k_{m,i}^{\top}
}
]
其中：
[
\Lambda_m=\exp(A_m\Delta_m)
]
[
A_m=-\operatorname{softplus}(dd_A_m)
]
[
\Delta_m=\operatorname{softplus}(dd_dt_m+b_\Delta)
]
________________________________________
10. Token 级读出与输出 gate
chunk 内所有 token 共享更新后的状态 (S_m)，但每个 token 仍然有自己的 (C/q)、(v)、(z)。
读出：
[
y_{m,i}
S_m q_{m,i}
+
D v_{m,i}
]
输出 gate 保留原版风格：
[
o_{m,i}
\operatorname{silu}(z_{m,i})
\odot
y_{m,i}
]
即：
[
\boxed{
o_{m,i}
\operatorname{silu}(z_{m,i})
\odot
\left(
S_mq_{m,i}
+
Dv_{m,i}
\right)
}
]
如果开启原版 outproj norm，则先 norm/gated norm 再 out projection；如果关闭，则直接：
[
y_{\text{out}}=W_o o
]
________________________________________
11. Norm / Derf 开关
mamba3-RE 原版里至少有：
•	(B) norm；
•	(C) norm；
•	optional out projection norm；
•	gated RMSNorm 相关路径。
新版要求：
在原版任何 norm 位置上添加开关，可以选择保留原版 norm，也可以改为 Derf。
可以抽象为：
[
\operatorname{NormSwitch}(x)=
\begin{cases}
\operatorname{RMSNorm}(x), & \text{use_derf_norm=False}\
\operatorname{Derf}(x), & \text{use_derf_norm=True}
\end{cases}
]
所以：
[
B \leftarrow \operatorname{NormSwitch}_B(B)
]
[
C \leftarrow \operatorname{NormSwitch}_C(C)
]
out projection norm 位置：
[
y \leftarrow \operatorname{NormSwitch}_{out}(y,z)
]
需要注意的是，Derf 不是严格 normalization，而是 norm replacement / elementwise saturating transform。因此最好每个位置独立开关，而不是全局一刀切：
use_derf_BC_norm
use_derf_out_norm
use_derf_pre_norm
________________________________________
12. 与 mamba3-RE 的差异清单
模块	mamba3-RE 原版	新版算子
dd_dt / (\Delta)	token 级，由 in_proj 得到	chunk 级，由卷积块特征得到
dd_A / (A)	token 级，由 in_proj 得到	chunk 级，由卷积块特征得到
trap	保留并传入 kernel	删除
data-based RoPE / angle	保留，输入生成 angle	删除
chunk 内写入	原版 token 级 SSM 动力学	chunk 内独立 gate 加权可加写入
写入 gate	无单独 chunk-write gate	(g_i=\operatorname{softplus}(r_i+b_g)) 或 sigmoid
(B/C) norm	RMSNormGated	加开关，可换 Derf
out norm	optional RMSNormGated	加开关，可换 Derf
MIMO/SISO 主结构	保留	基本保留
(D) skip	保留	保留
(z) gate	保留	保留
out projection	保留	保留

13. 最终极简公式
对每个 chunk (m)、每个 token (i\in\mathcal C_m)：
[
z_{m,i},v_{m,i},k_{m,i},q_{m,i},r_{m,i}
W_{\text{tok}}x_{m,i}
]
其中：
[
k_{m,i}\equiv B_{m,i}
]
[
q_{m,i}\equiv C_{m,i}
]
chunk 级特征：
[
f_m=\operatorname{Conv}_{L,L}(x)_m
]
chunk 级动态参数：
[
\Delta_m
\operatorname{softplus}(W_\Delta f_m+b_\Delta)
]
[
A_m
-\operatorname{softplus}(W_A f_m)
]
[
A_m
\operatorname{clamp}(A_m,\max=-A_{\text{floor}})
]
token 写入 gate：
[
g_{m,i}
\operatorname{softplus}(r_{m,i}+b_g)
]
chunk 写入项：
[
U_m
\frac{1}{L}
\sum_{i\in\mathcal C_m}
g_{m,i}
v_{m,i}
k_{m,i}^{\top}
]
chunk 级状态递推：
[
\boxed{
S_m
\exp(A_m\Delta_m)S_{m-1}
+
\Delta_m U_m
}
]
token 输出：
[
\boxed{
o_{m,i}
\operatorname{silu}(z_{m,i})
\odot
\left(
S_mq_{m,i}
+
Dv_{m,i}
\right)
}
]
最终输出：
[
y_{m,i}
W_o o_{m,i}
]
________________________________________
14. 一句话总结新版
新版算子是一个 mamba3-RE 最小侵入式图像化改造：
保留原版 Mamba3 的 (z,x/B/C,D,out_proj) 主体，但删除 trap 和 data-based RoPE；将 dd_dt 与 dd_A 从 token 级投影改为由 chunk 卷积特征生成的块级动态参数；chunk 内不再做 token 级因果动力学，而是用 token 独立写入 gate 对 (v_i k_i^\top) 做非因果可加聚合，chunk 间再执行一次 SSM 状态递推；所有原版 norm 位置增加 Derf 替换开关。
